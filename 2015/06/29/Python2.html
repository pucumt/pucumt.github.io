Python1- Features

29 June, 2015

Python is more and more famous now, but I don't have much time to use it in my normal dev. works, because I think it is hard to use to compare with C#, someone may say there are more libraries in Python than in C#. Yes, but still like C# more, because:

  1. C# has a very wonderful IDE, use VS you could find the library all classes, all public methods very easy, but how about python?

  2. I don't like Python thread mechanism, when the child thread is running, you cannot stop the parent process(when without setDaemon).

  3. I like the C# paragraph, once with brackets, it is easy to expend or close, I don't think Python can do it(the code paragraph inside methods).

  4. C# need reflect to see the code after compile, but Python code is open source, I do like Python this feature, but this is why Python is slow.

  5. Windbg is very easy to dubug the core and system memory, but how about Python?

  6. C# could compile to machine code(windows), how about Python?

  7. It is very easy to code both winform and webform in C#, how about Python?

  8. I am very sure which library is official and stable, but in Python, so many libraries, which one is the correct one you wanted?

Of course there are some features in Python is very well either:

  1. Cross platform, it is very easy to run both in windows and linux, I don't know why .net don't support it for so many years.

  2. Python is open source, it is very useful to people who need learn a language, especially to the guys without coding experience.

  3. Huge of libraries, I do like the libraries, it is very easy to solve some difficult things.

  4. Force every method is small and every each code level should indent.
