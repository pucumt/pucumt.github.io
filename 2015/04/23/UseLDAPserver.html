Use LDAP server to authenticate user identity

23 April, 2015

In the group company, the employees' account and password are all recorded in LDAP server, and usually there are SSO login needed, or the apps need login with the account exists in LDAP server. About the basic technology of LDAP server, you could search it from internet. I just wanted to talk about how to use it to authenticate a user login. And if your IT is really good at LDAP server and could told you about the LDAP structure, please ignore this article.

My situation

  1. I don't know the base DN, and I don't know where to get that.
  2. I know the LDAP server address and port, and lucky I get one of the LDAP server acccount could be used to search infos.

The steps to get LDAP structure:

  1. Download a LDAP server management tools(like Softerra LDAP Administrator) By default, it will login some LDAP servers, I am not if this only exists in Softera LdAP Administrator or not.

  2. Create a new profile to the LDAP server(notice the port).

  3. Click next and input the account that could search in LDAP server Usually the user will be cn=XXX or uid=XXX, that's both OK.

  4. After input account, the LDAP structure will be displayed. And here is my LDAP structure:
